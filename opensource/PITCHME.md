### 開源的缺點
1. 客群設定不容易聚焦。
2. 客製化功能不一定能快速開發。
3. 無法設立獨家標準。
4. 無法進行純軟體販賣商業模式。
5. 導入社群管理後，單方面私有利益可能減損。

## 沒有人是局外人

+++

### 讓每個人都成為"局內人"
- 重視每一個人的貢獻。
- 以意見領袖為管理者，確保效率最大化。
- 人人都是意見領袖，平行決策。

+++

### 用貢獻取代監督
- 有問題，沒人做，提出 Pull Request (Github)。
- 降低監督比例 = 減少人力資源浪費。
- 建立以「加分」為主的環境。
